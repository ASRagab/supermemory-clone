## Executive Summary

Two tests are failing due to **error message format mismatch** between test expectations and actual error messages generated by the `NotFoundError` class. The tests expect a simple format (`"Job {id} not found"`) but the error class produces a formatted message (`"Job with ID '{id}' not found"`).

This is a **test specification issue**, not a code logic issue. The queue system is functioning correctly.

## Root Cause Analysis

### Source of Error Messages

Both failures stem from the same root cause in the error handling chain:

**File:** `src/utils/errors.ts` (lines 215-231)
**Class:** `NotFoundError`

```typescript
export class NotFoundError extends AppError {
  readonly resourceType: string;
  readonly resourceId?: string;

  constructor(
    resourceType: string,
    resourceId?: string,
    code: ErrorCodeType = ErrorCode.NOT_FOUND
  ) {
    const message = resourceId
      ? `${resourceType} with ID '${resourceId}' not found`
      : `${resourceType} not found`;
    super(message, code);
    this.name = 'NotFoundError';
    this.resourceType = resourceType;
    this.resourceId = resourceId;
  }
}
```

### How Errors Are Generated

**File:** `src/queues/index.ts`

#### For moveToDeadLetterQueue (lines 220-230):

### The Mismatch Explanation

The `NotFoundError` class formats messages with the pattern:
```
"{resourceType} with ID '{resourceId}' not found"
```

But the tests expect:
```
"{resourceType} {resourceId} not found"
```

This is a **specification mismatch**: the actual error class implementation doesn't match the test's expected format.
